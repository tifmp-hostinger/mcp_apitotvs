RM-API — referência de endpoints

Integração com o TOTVS RM via SOAP. Ajuste a URL base, a chave e os campos; cada endpoint gera o cURL pronto (terminal ou Import cURL do n8n) e pode ser executado direto daqui com resposta na tela.
Base path na raiz: as rotas são /pessoas, /rm/test etc., sem /api. Autenticação: se o servidor tiver API_KEY, preencha o campo acima — o header X-API-Key entra nos cURLs e no Executar. Executar: roda a requisição do seu navegador. Servida desta mesma origem não há CORS; para outra base, configure CORS_ALLOWED_ORIGINS no servidor. POST grava dados reais no RM — a página pede confirmação antes de executar (exceção: corpo com "DRY_RUN": true, que não envia nada ao RM — e é o default do exemplo da baixa).
